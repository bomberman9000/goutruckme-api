# ⚖️ AI-Юрист — Автоматическая модерация претензий и форума

## 🤖 Как работает AI-Юрист модератор

### 📋 Претензии

**При создании претензии AI-Юрист автоматически:**

1. **Анализирует текст:**
   - Ищет ключевые слова высокой серьёзности
   - Определяет тип нарушения
   - Проверяет обоснованность

2. **Проверяет нарушителя:**
   - Рейтинг пользователя
   - Количество предыдущих претензий
   - Уровень доверия

3. **Принимает решение:**
   - **Критический риск (70-100):** Автоматически подтверждает, снимает баллы
   - **Высокий риск (50-70):** Автоматически подтверждает, снимает баллы
   - **Средний риск (30-50):** Отправляет на рассмотрение
   - **Низкий риск (<30):** Обычное рассмотрение

### 💬 Форум

**При создании поста AI-Юрист автоматически:**

1. **Проверяет содержание:**
   - Оскорбления
   - Клевета
   - Наличие доказательств
   - Качество текста

2. **Принимает решение:**
   - **Низкий риск (<30):** Автоматически верифицирует и публикует
   - **Средний риск (30-50):** Отправляет на модерацию
   - **Высокий риск (50+):** Блокирует до проверки

## 🎯 Автоматические действия

### Претензии:

- **Критический риск:** Статус → "reviewed", автоматическое снятие баллов, ответ AI
- **Высокий риск:** Статус → "reviewed", автоматическое снятие баллов
- **Остальные:** Статус → "pending", ожидание администратора

### Форум:

- **Низкий риск:** is_verified → true, пост публикуется сразу
- **Средний/Высокий риск:** is_verified → false, требуется модерация

## 🧩 Техническая спецификация (MVP)

### 1. Нормализация скоринга

Итоговый риск‑скор считается по формуле:

```
score = clamp(sum(feature_weights), 0, 100)
```

Где:

- `feature_weights` — сумма весов всех сработавших признаков
- `clamp(x, 0, 100)` — ограничение диапазона 0…100

Правила:

- диапазоны в таблицах трактуются так:
  - «+10–30» → +10 за каждый повторяющийся фактор, максимум +30
- все веса суммируются
- итог всегда приводится к диапазону 0…100

Смягчающие признаки (уменьшают риск):

| Признак | Баллы |
|------|-----|
| Приложены доказательства (фото/файлы/документы) | −15 |
| Текст структурирован и > 200 символов | −5 |
| Высокий trust level пользователя | −10 |

Минимальный риск после всех вычетов — 0.

### 3. Политика решений (претензии)

| Уровень | Действие |
|------|--------|
| Низкий | status = pending |
| Средний | status = pending |
| Высокий | freeze + отправка модератору |
| Критический | freeze + приоритетная модерация |

Важно:

Для категорий «мошенничество», «клевета», «уголовные обвинения»:
AI **никогда** не переводит претензию в подтверждённую автоматически — только `freeze + manual review`.

### 7. Аудит решений AI

Для каждого AI‑решения сохраняется:

- entity_type, entity_id
- score
- risk_level
- decision
- signals
- thresholds
- model_version
- created_at

Запись обязательна и используется для разборов и апелляций.

---

### 8. Апелляции

Поддерживается повторная проверка:

- статус претензии → `appeal_requested`
- повторный запуск модели с новой версией
- ручное решение администратора фиксируется отдельно

AI‑решение не может быть финальным для юридически чувствительных категорий.
