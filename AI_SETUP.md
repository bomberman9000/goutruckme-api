# 🤖 Настройка Искусственного Интеллекта для ГрузПоток

## 📋 Что нужно для работы ИИ

Для полноценной работы 5 ИИ-модулей нужно настроить один из LLM-провайдеров:

### ✅ Вариант 1: Локальный режим (без LLM) - РАБОТАЕТ СЕЙЧАС

По умолчанию ИИ работает в **локальном режиме** (mock mode) - это означает:
- ✅ Базовый анализ заявок работает
- ✅ Проверка реквизитов работает
- ✅ Анализ цены работает
- ❌ Глубокий юридический анализ через LLM - не используется
- ❌ Умные рекомендации через LLM - не используются

**Это нормально для начала!** Локальный режим уже работает.

---

### 🚀 Вариант 2: OpenAI GPT (Рекомендуется для продакшена)

**Преимущества:**
- ✅ Самый мощный ИИ
- ✅ Лучшее качество анализа
- ✅ Поддержка русского языка

**Как настроить:**

1. Получите API ключ: https://platform.openai.com/api-keys
2. Создайте файл `.env` в корне проекта:

```bash
# Включить LLM
AI_USE_LLM=true

# OpenAI
OPENAI_API_KEY=sk-your-key-here
OPENAI_MODEL=gpt-4o-mini
```

3. Перезапустите сервер

**Стоимость:** ~$0.15 за 1M токенов (gpt-4o-mini)

### 🇷🇺 Вариант 3: Yandex GPT (Российский аналог)

## 📝 Создание .env файла

Создайте файл `.env` в корне проекта (`gruzpotok/.env`):

```bash
# База данных
DATABASE_URL=sqlite:///./gruzpotok.db

# Безопасность
SECRET_KEY=your-secret-key-change-in-production

# ========== ИИ НАСТРОЙКИ ==========

# Включить LLM (true/false)
AI_USE_LLM=false

# Fallback на локальный анализ при ошибках
AI_FALLBACK_TO_LOCAL=true

# Таймаут запросов (секунды)
AI_TIMEOUT_SECONDS=30

# OpenAI (выберите один из провайдеров)
OPENAI_API_KEY=
OPENAI_MODEL=gpt-4o-mini

# Yandex GPT
YANDEX_GPT_KEY=
YANDEX_FOLDER_ID=

# GigaChat
GIGACHAT_KEY=
```

## 🛠️ Устранение проблем

### Проблема: "LLM не работает"

**Решение:**
1. Проверьте, что `AI_USE_LLM=true` в `.env`
2. Проверьте, что API ключ указан правильно
3. Проверьте интернет-соединение
4. Если ошибка - включите `AI_FALLBACK_TO_LOCAL=true` (будет работать локальный режим)

### Проблема: "Таймаут запросов"

**Решение:**
Увеличьте `AI_TIMEOUT_SECONDS=60` в `.env`

### Проблема: "Ошибка API ключа"

**Решение:**
1. Проверьте правильность ключа
2. Проверьте баланс аккаунта (для OpenAI)
3. Проверьте лимиты API
