# Deals System

Этот файл — документация по модулю **Deals** (сделки) в GouTruckMe.

> Важно: сюда **не** должен попадать `docker-compose.yml` или любые YAML‑фрагменты. Если видишь тут `services:` — файл был случайно перезаписан.

## Назначение

Deals (сделки) — это слой между грузом (Load/Cargo) и фактом выполнения/закрытия работы.

Типовой сценарий:
1. Пользователь выбирает груз.
2. Создаётся сделка со статусом `IN_PROGRESS`.
3. Далее сделка подтверждается/закрывается/отменяется.

## Статусы сделки

Минимальный набор (пример):
- `IN_PROGRESS` — работа в процессе
- `CONFIRMED` — подтверждено
- `COMPLETED` — завершено
- `CANCELLED` — отменено

> Источник истины для «активности» — функция `isActiveStatus()` (активные: `IN_PROGRESS`, `CONFIRMED`).

## Связь с грузом

- В сделке хранится `cargoId` (ID груза).
- Дополнительно хранится `cargoSnapshot` — **снимок груза на момент создания сделки**.

Это нужно, чтобы документы/печать/история работали даже если груз изменился или исчез из текущего списка.

### CargoSnapshot

Требования к снапшоту:
- только примитивы и строки (никаких вложенных объектов/Date)
- поля дат — строками ISO
- есть `snapshotVersion` и `snapshotAt`

## UI/Документы

Во всех местах, где требуется груз из сделки:
- сначала ищем груз в актуальном списке
- если не найден — используем `cargoSnapshot`

Отображение источника данных:
- `__source = 'live'` — актуальные данные
- `__source = 'snapshot'` — данные из сделки
