## 📁 Где находится

### Основной файл
- **`app/static/index.html`** — весь код (HTML + JavaScript)

### Структура кода в index.html

#### HTML структура (строки ~109-310)
- Секция `#page-loads` — страница грузов
- Панель фильтров `#filtersPanel`
- Таблица с заголовками
- Контейнеры для активных фильтров и пагинации

#### JavaScript функции (строки ~647-1900)

**Генерация данных:**
- `generateMockLoads(count)` — создаёт мок-данные

**Состояние:**
- `cargoState` — объект состояния
- `loadCargoState()` — загрузка из localStorage
- `saveCargoState()` — сохранение в localStorage

**Фильтрация:**
- `filterLoads(loads, filters)` — фильтрация грузов
- `applyFilter(key, value)` — применение фильтра
- `debounceFilter(key, value)` — debounce для текстовых полей
- `clearAllFilters()` — сброс всех фильтров
- `removeFilter(key)` — удаление одного фильтра
- `getActiveFilters(filters)` — получение активных фильтров
- `validateFilters(filters)` — валидация фильтров

**Сортировка:**
- `sortLoads(loads, column, order)` — сортировка
- `sortTable(column)` — обработчик клика по заголовку
- `updateSortIcons()` — обновление иконок сортировки

**Рендеринг:**
- `renderTable()` — рендеринг таблицы
- `renderActiveFilters()` — рендеринг чипсов фильтров
- `renderPagination()` — рендеринг пагинации
- `applyFiltersAndSort()` — применение всех изменений

**Форматирование:**
- `formatPrice(price)` — форматирование цены
- `formatDate(dateString)` — форматирование даты
- `formatTime(dateString)` — форматирование времени
- `formatDistance(distance)` — форматирование расстояния

**Создание груза:**
- `createLoad()` — создание нового груза
- `validateLoadForm()` — валидация формы
- `calculatePricePerKm()` — расчёт ставки за км
- `formatPhone(input)` — форматирование телефона
- `calculateRiskLevel(load, allLoads)` — расчёт риска

**Утилиты:**
- `setPage(page)` — установка страницы
- `setItemsPerPage(count)` — установка элементов на странице
- `toggleFiltersCollapse()` — сворачивание фильтров
- `showNotification(message, type)` — уведомления

**Загрузка данных:**
- `loadLoads()` — загрузка грузов (API или моки)

## 🧪 Как протестировать

### 1. Запуск приложения

```bash
cd /Users/mac/Projects/atinew/gruzpotok
source venv/bin/activate
uvicorn app.api.main:app --host 0.0.0.0 --port 8080 --reload
```

Откройте в браузере: `http://localhost:8080`

### 2. Тестирование фильтров

1. **Текстовые фильтры:**
   - Введите "Москва" в поле "Откуда"
   - Введите "Казань" в поле "Куда"
   - Проверьте, что фильтрация работает с задержкой (debounce)

2. **Числовые фильтры:**
   - Установите вес: от 5 до 15 т
   - Установите объём: от 10 до 30 м³
   - Проверьте валидацию (от не может быть больше до)

3. **Дата:**
   - Выберите дату в прошлом → должна появиться ошибка
   - Выберите дату в будущем → должно работать

4. **Тип кузова:**
   - Выберите "Тент" из списка
   - Проверьте фильтрацию

5. **Активные фильтры:**
   - После применения фильтров должны появиться чипсы над таблицей
   - Клик по ✕ на чипсе должен удалить фильтр

6. **Сброс:**
   - Нажмите "Сбросить" → все фильтры должны очиститься

### 3. Тестирование сортировки

1. Кликните на заголовок "Ставка / за км"
   - Первый клик: сортировка по возрастанию (↑)
   - Второй клик: сортировка по убыванию (↓)
   - Третий клик: сброс сортировки (⇅)

2. Проверьте другие колонки:
   - Дата / время
   - Дистанция
   - Вес

### 4. Тестирование пагинации

1. Измените "Показано" на 20/50/100
2. Проверьте навигацию ← →
3. Проверьте отображение "X–Y из Z"

### 5. Тестирование создания груза

1. Нажмите "➕ Добавить груз"
2. Заполните форму:
   - Откуда: Москва
   - Куда: Казань
   - Вес: 10 т
   - Объём: 20 м³
   - Тип кузова: Тент
   - Дата погрузки: завтра
   - Ставка: 25000 ₽
   - Дистанция: 800 км
   - Телефон: +7 999 123-45-67
3. Проверьте:
   - Автосчёт ставки за км (должно появиться ~31 ₽/км)
   - Форматирование телефона при вводе
4. Нажмите "Создать груз"
5. Проверьте:
   - Груз появился в списке
   - Уведомление "✅ Груз успешно создан!"
   - Форма очистилась

### 6. Тестирование валидации

1. Попробуйте создать груз без обязательных полей → должны появиться ошибки
2. Укажите одинаковые города → должна появиться ошибка
3. Укажите дату в прошлом → должна появиться ошибка
4. Укажите отрицательную ставку → должна появиться ошибка

### 7. Тестирование сохранения состояния

1. Примените фильтры
2. Измените сортировку
3. Измените количество элементов на странице
4. Сверните панель фильтров
5. Обновите страницу (F5)
6. Проверьте, что всё восстановилось

### 8. Тестирование риск-бейджей

1. Создайте груз с очень низкой ставкой (например, 10000 ₽ на 800 км)
2. Проверьте, что риск-бейдж показывает "Высокий" или "Средний"
3. Наведите на бейдж → должен появиться tooltip с причинами

### 9. Тестирование пустого состояния

1. Примените фильтры, которые не дают результатов
2. Проверьте красивое пустое состояние с кнопкой "Добавить груз"

### 10. Тестирование sticky header

1. Прокрутите таблицу вниз
2. Проверьте, что заголовок остаётся видимым

## 🎯 Особенности реализации

### Без внешних API
- ✅ Все функции работают локально
- ✅ Мок-данные генерируются на клиенте
- ✅ Расчёт риска — простой алгоритм без AI
- ✅ Нет зависимости от платных сервисов

### Готовность к будущему AI
- ✅ Структура данных позволяет легко добавить AI-модули
- ✅ Risk badges готовы к интеграции с AI-Antifraud
- ✅ Поля формы готовы к автозаполнению через AI

### Производительность
- ✅ Debounce для текстовых фильтров
- ✅ Локальное кеширование состояния
- ✅ Эффективная фильтрация и сортировка

## 📝 Примечания

- Мок-данные генерируются при каждой загрузке страницы
- При наличии API токена грузы отправляются на сервер, но также добавляются локально
- Состояние сохраняется в localStorage (может быть очищено пользователем)
- Все функции работают без подключения к бэкенду
