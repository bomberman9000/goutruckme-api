### 3️⃣ Умная кнопка "Взять"

**Три варианта:**

1. **Low риск:**
   - Обычная кнопка (градиент purple-blue)
   - Немедленное действие

2. **Medium риск:**
   - Secondary стиль (dark-border с жёлтой границей)
   - Tooltip при наведении: "⚠️ Проверь условия"
   - Рекомендация изучить детали

3. **High риск:**
   - Outline стиль (прозрачная с красной границей)
   - Подтверждение через `confirmHighRiskBid()`
   - Модальное окно с причинами риска
   - Пользователь видит все причины перед подтверждением

**Функции:**
- `addBid(loadId)` — для low/medium
- `confirmHighRiskBid(loadId)` — для high с подтверждением

**Где находится:**
- `app/static/index.html` строки ~1409-1429 (генерация кнопок)
- `app/static/index.html` строки ~2140-2168 (функции)

## 📊 Логика аномалий цены

**Расчёт:**
1. Находим похожие грузы (дистанция ±200км)
2. Вычисляем медиану ставки за км
3. Сравниваем текущую цену с медианой
4. Если отклонение ≥30% → аномалия

**Визуализация:**
- Низкая цена → жёлтая подсветка + ⚠️
- Высокая цена → зелёная (нормально)

## 🎨 UX-психология

**Почему это работает:**
1. **Визуальная иерархия** — глаз цепляется за аномалии
2. **Предупреждение, не запрет** — взрослая позиция
3. **Контроль** — чипсы показывают, что применено
4. **Понятность** — tooltip объясняет причины
5. **Дорогой продукт** — детали имеют значение

## 🧪 Как протестировать

1. **Tooltip:**
   - Наведите на бейдж риска
   - Должен появиться tooltip с причинами

2. **Аномалии цены:**
   - Найдите груз с подозрительно низкой ценой
   - Должна быть жёлтая подсветка и ⚠️

3. **Умная кнопка:**
   - Low риск → обычная кнопка
   - Medium риск → secondary + tooltip
   - High риск → outline + подтверждение

4. **Чипсы:**
   - Примените фильтры
   - Чипсы должны появиться над таблицей
   - Клик по ✕ должен удалить фильтр

5. **Пустое состояние:**
   - Примените фильтры, которые не дают результатов
   - Должно быть пустое состояние с кнопками
